# MCP Server Package Templates (FastMCP 3.0 + uv)

Replace `{{SERVER_NAME}}` with the kebab-case server name (e.g., `weather`).
Replace `{{SERVER_NAME_UNDERSCORE}}` with the underscore form (e.g., `weather`).

This monorepo uses [uv](https://docs.astral.sh/uv/) for Python dependency management. Each package has a `pyproject.toml` + `uv.lock` rather than `requirements.txt`.

## app.py

```python
"""
{{SERVER_NAME}} MCP Server

FastMCP 3.0 server with Streamable HTTP transport for AWS Lambda.
"""

import os

from dotenv import load_dotenv
from fastmcp import FastMCP
from fastmcp.exceptions import ToolError

# Load .env for local dev. Silently a no-op in Lambda (no .env file exists there).
load_dotenv()


def create_mcp_server() -> FastMCP:
    """Create a fresh FastMCP server instance."""
    mcp = FastMCP(
        name="{{SERVER_NAME_UNDERSCORE}}-server",
        instructions="""Description of what this MCP server does.

Provide helpful context for AI assistants using these tools.""",
    )

    # =========================================================================
    # Tool: example_tool
    # =========================================================================

    @mcp.tool
    async def example_tool(query: str, limit: int = 10) -> dict:
        """
        Example tool. Replace with your actual implementation.

        Args:
            query: The search query
            limit: Max results to return (default: 10)
        """
        if not query or len(query.strip()) < 2:
            raise ToolError("Query must be at least 2 characters")

        return {
            "query": query,
            "limit": limit,
            "results": [{"id": 1, "title": f"Result for: {query}"}],
            "total": 1,
        }

    return mcp


# =============================================================================
# Lambda Handler
# =============================================================================

def lambda_handler(event, context):
    """AWS Lambda handler."""
    from mangum import Mangum

    mcp = create_mcp_server()
    return Mangum(mcp.http_app(), lifespan="on")(event, context)


# =============================================================================
# Local Development
# =============================================================================

if __name__ == "__main__":
    import uvicorn

    mcp = create_mcp_server()
    port = int(os.environ.get("PORT", 8000))
    uvicorn.run(mcp.http_app(), host="0.0.0.0", port=port)
```

## pyproject.toml

```toml
[project]
name = "{{SERVER_NAME}}"
version = "0.1.0"
description = "{{SERVER_NAME}} MCP server"
requires-python = ">=3.12,<3.13"
dependencies = [
    "fastmcp>=3.0",
    "mangum>=0.17.0",
    "uvicorn>=0.30.0",
    "python-dotenv>=1.0",
    "httpx>=0.27.0",
]

[tool.uv]
# This is an application, not a library — uv should not try to install it as a package.
package = false
```

Add server-specific deps to `[project].dependencies` as needed (e.g., `"boto3>=1.34.0"`, `"opensearch-py>=2.4.0"`).

`python-dotenv` is included so `app.py` can load `.env` automatically during local dev. It's a no-op in Lambda where no `.env` exists, so it's safe to keep in the production image.

## uv.lock

**Do not write this file by hand.** It's generated by `uv lock` (or implicitly by `uv sync`) the first time you install dependencies, and committed to git for reproducible builds.

```bash
cd packages/{{SERVER_NAME}}
uv lock        # creates uv.lock
git add uv.lock
```

The Dockerfile uses `uv.lock` with `--frozen` to guarantee CI/Lambda installs the exact same versions as local dev.

## Dockerfile

Multi-stage build: uv exports the lockfile to requirements format in the builder stage and installs into `${LAMBDA_TASK_ROOT}`, then the final stage just copies the result and the app code.

```dockerfile
# Stage 1: builder — uses uv to install deps into LAMBDA_TASK_ROOT
FROM ghcr.io/astral-sh/uv:latest AS uv

FROM --platform=linux/amd64 public.ecr.aws/lambda/python:3.12 AS builder
ENV UV_COMPILE_BYTECODE=1
ENV UV_LINK_MODE=copy

RUN --mount=from=uv,source=/uv,target=/bin/uv \
    --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \
    --mount=type=bind,source=pyproject.toml,target=pyproject.toml \
    uv export --frozen --no-dev --no-editable -o /tmp/requirements.txt && \
    uv pip install -r /tmp/requirements.txt --target "${LAMBDA_TASK_ROOT}"

# Stage 2: runtime — copy installed deps + app code
FROM --platform=linux/amd64 public.ecr.aws/lambda/python:3.12
COPY --from=builder ${LAMBDA_TASK_ROOT} ${LAMBDA_TASK_ROOT}
COPY app.py ${LAMBDA_TASK_ROOT}/
# Add any other source files here:
# COPY my_client.py ${LAMBDA_TASK_ROOT}/

CMD ["app.lambda_handler"]
```

**Pinning uv:** Replace `:latest` with a specific version (e.g., `ghcr.io/astral-sh/uv:0.5.0`) once the project is in production — `:latest` is fine for early scaffolding but means image builds aren't fully reproducible.

The `--mount=type=cache` lines accelerate rebuilds; the `--mount=type=bind` lines mean `uv.lock` / `pyproject.toml` aren't copied into the final image, only used during install.

## .env

Gitignored (matched by the repo-root `.gitignore`). Loaded automatically by `app.py` via `python-dotenv` when running locally.

```
# Server-specific environment variables for local dev
# EXAMPLE_API_BASE_URL=https://api.example.com
# EXAMPLE_API_KEY=local-dev-key
```

In Lambda, these values come from the stack's `environment:` prop in `infrastructure.ts` (and ultimately from GitHub Actions `vars`/`secrets`) — no `.env` file is needed there.

## Adding More Tools

Add inside `create_mcp_server()`:

```python
@mcp.tool
async def search_items(query: str, category: str | None = None, limit: int = 20) -> dict:
    """Search for items matching the query."""
    if not query:
        raise ToolError("Query is required")

    results = await perform_search(query, category, limit)
    return {"query": query, "category": category, "results": results, "total": len(results)}
```

Note FastMCP 3.0 uses `@mcp.tool` (no parentheses).

## Error Handling

Use `ToolError` for messages the client should see. `ToolError` messages are always sent to clients regardless of `mask_error_details`.

```python
from fastmcp.exceptions import ToolError

@mcp.tool
async def my_tool(param: str) -> dict:
    if not param:
        raise ToolError("Parameter is required")
    try:
        return await external_api_call(param)
    except ExternalAPIError as e:
        raise ToolError(f"API call failed: {e}")
```

## Testing Locally

```bash
cd packages/{{SERVER_NAME}}

# First run: install deps. Subsequent runs of `uv sync` are near-instant if the lockfile hasn't changed.
uv sync

# Run the server. `uv run` activates the project's .venv automatically.
PORT=8000 uv run python app.py

# In another shell:
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'
```

`load_dotenv()` at the top of `app.py` reads `packages/{name}/.env` on startup, so any value you set there is visible via `os.environ.get(...)` in your tools.
