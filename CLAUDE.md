# Claude Context

A public-template MCP server monorepo. Python FastMCP 3.0 servers, managed with [uv](https://docs.astral.sh/uv/), packaged in Docker containers and deployed to AWS Lambda via CDK and GitHub Actions.

## Architecture

- **Python + FastMCP 3.0**: Use `@mcp.tool` decorator, `create_mcp_server()` factory, `lambda_handler()` entry point
- **uv**: Dependency management per package (`pyproject.toml` + `uv.lock` + `.python-version`)
- **Docker Lambda**: `DockerImageFunction` in CDK, images stored in ECR. Dockerfile uses uv to install deps into `${LAMBDA_TASK_ROOT}`.
- **Mangum adapter**: ASGI adapter that wraps `mcp.http_app()` for Lambda

## Key Patterns

**Package naming:** `packages/{name}/` where `name` is kebab-case.

**Server structure:** Each server in `packages/{name}/`:
- `app.py` — FastMCP server with tools defined via `@mcp.tool`. Calls `load_dotenv()` so `.env` is auto-loaded in local dev.
- `pyproject.toml` — Project metadata + dependencies (`fastmcp>=3.0`, `mangum`, `uvicorn`, `python-dotenv`). Marked `[tool.uv] package = false` (it's an app, not a library).
- `.python-version` — Pins to `3.12` so uv auto-selects (and auto-downloads) the matching interpreter.
- `uv.lock` — Generated by uv, committed for reproducible builds.
- `Dockerfile` — Multi-stage Lambda container that uses uv to install deps into `${LAMBDA_TASK_ROOT}`.
- `.env` — Local dev environment variables (gitignored repo-wide).

**CDK stacks:** One stack per server using `McpDockerStack`. Register in `infrastructure/bin/infrastructure.ts`. Always include `imageTag` prop (uses git SHA via `IMAGE_TAG` env var) to ensure Lambda code updates on every deploy.

**ECR repos:** Named `mcp-{server-name}` (created automatically by the deploy workflow).

## Commands

```bash
# Local development (no Docker — uv with .env auto-loaded by app.py)
cd packages/{name}
uv sync                            # install deps from uv.lock; downloads Python 3.12 if missing
PORT=8000 uv run python app.py     # uv run auto-activates the project's .venv

# Add a new dependency
cd packages/{name}
uv add <package>                   # updates pyproject.toml AND uv.lock

# Deploy CDK stack manually
cd infrastructure && npx cdk deploy McpServer-{name}-dev
```

**uv install (one-time):** `curl -LsSf https://astral.sh/uv/install.sh | sh`

Docker is used only for the Lambda deploy (built and pushed by GitHub Actions). Local dev should not need Docker.

## FastMCP 3.0 Tool Pattern

```python
from fastmcp import FastMCP
from fastmcp.exceptions import ToolError

def create_mcp_server() -> FastMCP:
    mcp = FastMCP(name="server-name")

    @mcp.tool
    async def my_tool(query: str, limit: int = 10) -> dict:
        """Tool description for AI assistants."""
        if not query:
            raise ToolError("Query is required")
        return {"results": [...]}

    return mcp

def lambda_handler(event, context):
    from mangum import Mangum
    mcp = create_mcp_server()
    return Mangum(mcp.http_app(), lifespan="on")(event, context)
```

## Deployment Flow

GitHub Actions (`.github/workflows/deploy.yml`) on push to `main`:
1. Detects changed packages under `packages/**`
2. Creates ECR repo if missing
3. Builds and pushes the Docker image (tagged with git SHA)
4. Runs `cdk deploy McpServer-{name}-{stage}` with `IMAGE_TAG` set to the git SHA
